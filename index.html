HTML

  • The head element is a container for metadata about the webpage
  • Meta elements contain information for the browser about the webpage
  • The body element represents the visibile content shown to the user

CSS

  • A margin references the space between the outside of the element and display space available.
  • A padding references the space between content and the border of the division.
  • There are three ways to set CSS values: in-line, internal, and external. External seems to be the most efficient in assigning reducing redundant code.
  • CSS can set values for common HTML elements as well as create classes to values to a set of similar divisions. Classes must be referenced in HTML.

Git

  • git status: checks what branch we are currently on and provides information about files that have been changed and their commit status
  • git checkout -b branch-name: creates a new branch and switches to it
  • git workflow for starting a new branch: Navigate to working directory, git pull origin main (to update directory), git status to check what repo you are working from, git checkout -b feature/name-of-feature, begin work
  • git workflow for saving work for branch: git status to check you are on branch, git add -A to stage changes, git commit -m "commit message".
  • git workflow for PR: follow saving work workflow, git pull origin main to update current repo with any changes that have been pushed to main, git push origin feature/name-of-feature, navigate to branch on github and follow workflow for PR.

JavaScript

  • JavaScript creates functionality and interactibility for a webpage.
  • Variables are containers that hold information. Set by var =
  • Arrays hold multiple variables. Can be multiple types.
  • For loops repeat a section of code for a defineed length. Called by for (variable; a length; and a counter)
  • A function is a set of code that can be called repetitively. Only performs work when called.
